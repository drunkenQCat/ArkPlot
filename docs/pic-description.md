# 🖼️ 图片描述配置指南

ArkPlot 可以调用视觉模型为游戏中的立绘和场景图生成文字描述，让小说化时的 AI 能"看到"角色长相和场景氛围，从而写出更生动的文本。

## 为什么需要图片描述？

小说化时，LLM 只能看到剧本文本：

> 【char】阿米娅 【dialog】博士，请下达指令！

它不知道阿米娅长什么样、身处什么环境。图片描述模块会：

1. 提取该场景的立绘和背景图
2. 调用视觉模型生成描述（如"少女身后是昏黄的指挥室，她紧握着剑柄，目光坚定"）
3. 将描述注入小说文本中，让小说化模型有更多素材

## Provider 选型

| Provider | 成本 | 速度 | 质量 | 适用场景 |
|----------|------|------|------|----------|
| **百炼云端** | 按 token 计费 | 快（云端） | 高（qwen-vl-max） | 追求质量、不想折腾本地环境 |
| **Ollama 本地** | 免费 | 慢（本地推理） | 中（取决于模型） | 有显卡、不想花钱、隐私敏感 |
| **自定义** | 取决于平台 | 取决于平台 | 取决于模型 | 有其他 OpenAI 兼容视觉接口 |

**推荐**：
- 有 NVIDIA 显卡（8GB+ VRAM）→ **Ollama + qwen3-vl:8b**，免费且足够好用
- 无显卡或追求速度 → **百炼 + qwen-vl-max**，云端快速高质量

## Ollama 本地配置

### 1. 安装 Ollama

前往 [ollama.com](https://ollama.com) 下载安装。安装后终端运行：

```bash
ollama serve
```

确保 Ollama 服务在后台运行。

### 2. 拉取视觉模型

```bash
ollama pull qwen3-vl:8b
```

其他可选模型：
- `llava:7b` — 通用视觉模型，质量略低
- `qwen2.5-vl:7b` — 上一代 qwen 视觉模型

### 3. ArkPlot 配置

1. 打开 ArkPlot 设置页
2. 进入「图片描述设置」标签页
3. Provider 选择「Ollama」
4. 模型选择下拉框中应出现 `qwen3-vl:8b`（如果没出现，点击刷新按钮）
5. Base URL 默认 `http://localhost:11434`，一般无需修改

## 百炼云端配置

### 1. 获取 API Key

同小说化配置，在 [百炼控制台](https://bailian.console.aliyun.com/) 创建 API Key。

### 2. ArkPlot 配置

1. 打开 ArkPlot 设置页
2. 进入「图片描述设置」标签页
3. Provider 选择「百炼」
4. 模型选择 `qwen-vl-max` 或 `qwen-vl-plus`
5. API Key 从环境变量 `DASHSCOPE_API_KEY` 自动读取，或在设置页直接填写

## 启用图片描述

配置完成后：

1. 回到 ArkPlot 主窗口
2. 勾选「启用图片描述」
3. 点击「开始」

图片描述会在小说化之前自动执行。ArkPlot 会：
- 从 PRTS Wiki 或本地缓存提取立绘和背景图 URL
- 下载图片 → 调用视觉模型 → 生成描述
- 描述存入数据库缓存，同一图片不重复调用

## 系统提示词

图片描述使用专门的系统提示词，可在设置页「图片描述设置 → 系统提示词」中查看和修改。

默认提示词要求模型：
- **禁止元评论** — 不要说"这是一幅画"、"图中有一个女孩"
- **禁止风格分析** — 不要说"动漫风格"、"游戏美术"
- **叙事语言** — 用小说笔法描述环境氛围、关键物体、人物姿态
- **字数限制** — 控制在 200 字以内，简洁不啰嗦

你可以根据偏好修改提示词，比如要求"更关注服饰细节"或"更强调环境氛围"。

## 已知问题

**干扰图自动跳过**

部分图片是游戏引擎占位符，没有实际内容：
- `bg_black.png` — 纯黑背景
- `char_empty.png` — 空白角色

ArkPlot 会自动识别并跳过这些图片，不调用视觉模型。

**首次耗时较长**

首次运行时需要下载所有图片并调用视觉模型。一个活动可能有 20-50 张图片，Ollama 本地推理每张约 3-5 秒，百炼云端约 1-2 秒。后续重复运行会很快（缓存生效）。

**部分图片描述质量不佳**

极少数图片（如纯色背景、无角色立绘）可能生成无意义描述。你可以在设置页「图片描述设置」中点击「清空描述缓存」按钮，删除所有记录后重新运行小说化。或者用 CLI 命令 `arkplot clear-picdesc` 清空数据库。

## 常见问题

**Q: 图片描述会消耗小说化的 token 吗？**

A: 图片描述单独调用视觉模型，不消耗小说化模型的 token。但描述文本会注入小说源文件，小说化时会处理这些文本。

**Q: Ollama 提示 "model not found" 怎么办？**

A: 确保 `ollama pull qwen3-vl:8b` 已执行，且 Ollama 服务在运行。终端运行 `ollama list` 确认模型已拉取。

**Q: 百炼调用失败怎么办？**

A: 检查 API Key 是否正确设置（环境变量或设置页）。百炼有免费额度限制，超出后需充值。

**Q: 图片描述可以关闭吗？**

A: 可以。不勾选「启用图片描述」即可。小说化会正常执行，只是没有图片描述注入。
