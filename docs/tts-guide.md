# 🔊 TTS 语音生成指南

ArkPlot 可以将小说化后的剧情转化为多角色配音的有声小说 MP3。本文档介绍 TTS 管线原理、配置步骤和操作流程。

## TTS 管线原理

小说化生成的 `_novel.md` 是连贯的小说文本，没有角色分段标记。TTS 管线需要把它"拆回"按角色的片段，才能给不同角色分配不同音色：

```
小说文本（_novel.md）
    │
    ▼  LLM 对齐管线
按角色分段的片段表（SegmentRow[]）
    │
    ▼  音色匹配
每个片段 → 对应音色（男声/女声）
    │
    ▼  Edge TTS 合成
每个片段 → MP3 文件
    │
    ▼  FFmpeg 合并（可选）
同一角色连续片段 → 单个 MP3
```

### 对齐是什么？

"对齐"是把小说文本拆回按角色的片段。ArkPlot 使用**纯本地算法**：
1. 用 `DialogExtractor` 从小说文本提取所有引号内的对话
2. 从数据库加载原始剧情数据（有角色标记）
3. 按对话顺序匹配，生成片段表：`{角色名, 起始位置, 结束位置, 台词文本}`

**无需调用 LLM，速度快（通常 <1 秒）**。对齐结果缓存到本地 JSON 文件（`_align_cache/{hash}.json`），同一小说文件不重复计算。

### 音色如何分配？

每个角色自动分配 Edge TTS 音色：
- **性别推断**：从角色名（如"阿米娅"→女、"博士"→男）和文本中的"他/她"推断性别
- **默认音色**：女声 `zh-CN-XiaoxiaoNeural`，男声 `zh-CN-YunxiNeural`
- **手动调整**：音色配置表中可以手动选择其他音色

### MP3 如何生成？

- **单片段 MP3**：每个片段独立生成一个 MP3 文件（`_seg_001.mp3`）
- **角色合并 MP3**：同一角色连续对话会合并为单个 MP3（`阿米娅_ch01.mp3`）
- **整章 MP3**：点击「导出当前章节 MP3」会合并所有片段为单个文件

## 快速开始

### 第一步：生成小说文件

TTS 需要 `_novel.md` 文件作为输入。确保已完成 AI 小说化：

1. 主界面勾选「启用小说化生成」
2. 选择活动，点击「开始」
3. 等待小说化完成，输出目录出现 `_novel.md`

### 第二步：打开 TTS 窗口

主界面点击「TTS 语音生成」按钮，打开 TTS 专用窗口。

### 第三步：选择小说文件

左侧面板「小说文件」列表会显示所有 `_novel.md` 文件。选中要生成 TTS 的文件。

> 如果列表为空，说明还没有生成小说文件，请先完成第一步。

### 第四步：配置音色

左侧面板「音色配置」表格显示每个角色及其音色：

| 角色名 | 性别 | 当前音色 |
|--------|------|----------|
| 阿米娅 | 女 | zh-CN-XiaoxiaoNeural |
| 博士 | 男 | zh-CN-YunxiNeural |
| 凯尔希 | 女 | zh-CN-XiaoxiaoNeural |

点击音色下拉框可以手动调整。可选音色：

| 音色 ID | 性别 | 特点 |
|---------|------|------|
| `zh-CN-XiaoxiaoNeural` | 女 | 温柔知性，适合大多数女性角色 |
| `zh-CN-XiaoyiNeural` | 女 | 活泼甜美 |
| `zh-CN-YunxiNeural` | 男 | 沉稳青年男声 |
| `zh-CN-YunjianNeural` | 男 | 激情演讲风格 |
| `zh-CN-YunxiaNeural` | 男 | 亲切童声 |

### 第五步：生成整章

点击左侧面板「生成整章」按钮：

1. 对齐管线自动执行（纯本地算法，速度快）
2. 每个片段调用 Edge TTS 生成 MP3
3. 进度条显示当前进度，日志区输出详情

生成完成后，右侧面板 DataGrid 每行会出现独立的音频播放器控件。

### 第六步：预览和连播

**单片段预览**：点击 DataGrid 每行的播放按钮，预览该片段 MP3。

**连播模式**：点击右侧面板顶部的「▶ 从选中行开始连播」：
- 自动播放选中行 → 下一行 → 下一行...
- 当前播放行蓝色闪动高亮
- DataGrid 自动滚动跟随
- 场景 Gallery 同步切换（如果有场景图）

连播控制按钮：
- `⏮ 上一句`：跳到上一片段
- `下一句 ⏭`：跳到下一片段
- `■ 停止`：停止连播

### 第七步：导出 MP3

右侧面板底部有两个导出按钮：

- **导出当前章节 MP3**：合并当前章节所有片段为单个 MP3 文件
- **导出全部章节 MP3**：合并所有章节为单个 MP3 文件（适合一次性听完整个活动）

## 高级功能

### 右键生成选中行

DataGrid 支持多选（按住 Shift 点击多行）。选中多行后右键 →「生成选中行」，只生成选中片段的 MP3。适合：
- 只想听某几段对话
- 重新生成某个角色的音色调整后的片段

### 搜索片段

右侧面板搜索框可以搜索：
- 角色名（如输入"阿米娅"，只显示阿米娅的片段）
- 台词文本（如输入"博士"，显示所有包含"博士"的片段）

### 章节导航

右侧面板顶部章节选择器：
- 下拉框选择章节
- `◄` 上一章节按钮
- `►` 下一章节按钮

切换章节时会自动触发重新对齐（如果该章节小说文件变化）。

## 输出文件结构

TTS 输出目录结构：

```
output/无忧梦呓/
├── 无忧梦呓_novel_deepseek-v4-flash.md    # 小说源文件
├── tts/
│   ├── seg_001.mp3                         # 片段 MP3（按序号）
│   ├── seg_002.mp3
│   ├── ...
│   ├── 阿米娅_ch01.mp3                     # 角色合并 MP3
│   ├── 博士_ch01.mp3
│   ├── 无忧梦呓_ch01.mp3                   # 整章 MP3
│   └── 无忧梦呓_all.mp3                    # 全部章节 MP3
```

## 常见问题

**Q: 对齐失败怎么办？**

A: 对齐是纯本地算法，失败通常是数据问题：
- 检查小说文件是否生成（`_novel.md`）
- 检查数据库中是否有对应的原始剧情数据
- 查看日志区的错误信息（如"找不到匹配的对话"）

**Q: Edge TTS 调用失败？**

A: Edge TTS 是微软免费服务，偶尔会超时或限流。ArkPlot 会自动重试 3 次。如果持续失败：
- 检查网络连接
- 等待几分钟后重试
- 某些片段可能生成失败，可以右键单独重新生成

**Q: 音色不满意怎么换？**

A: 音色配置表中手动调整后，点击「生成整章」会重新生成所有片段。或者右键选中某角色的所有片段 →「生成选中行」，只重新生成该角色。

**Q: TTS 可以用自定义引擎吗？**

A: 当前版本只支持 Edge TTS。后续版本计划支持自定义 TTS 引擎（如 Azure TTS、本地 TTS 模型）。

**Q: 生成很慢怎么办？**

A: Edge TTS 每个片段约 1-2 秒。一个章节有 50+ 片段时，整章生成约 1-2 分钟。进度条和日志会实时显示进度。生成结果会缓存，重复运行只生成新增/修改片段。
